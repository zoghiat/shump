# Game Logic Plan

This document describes the game logic section of the Space Shooter project.

## Responsible Member
Member 1 is responsible for the main gameplay logic.

## Main Classes

- GameManager
- Player
- Enemy
- Bullet
- Coin
- PowerUp

## Game Logic Responsibilities

The game logic section includes:

- Player movement
- Player shooting
- Enemy movement
- Bullet movement
- Collision detection
- Score handling
- Coin collection
- PowerUp handling
- Wave progression
- Game over logic

## Object-Oriented Design

The game uses object-oriented programming principles.

The Enemy class is used as a base class for different enemy types.
Each enemy type has its own movement and behavior.

This helps us use inheritance and polymorphism in the game logic.
